## 0. 설계 원칙

**"올바르게 못 만들 거면 넣지 않는다."**
마감을 이유로 결함·임시 제약을 안고 가지 않는다. 채점 핵심(NOI·IRR·민감도)만 **검증된 무결점 상태**로 가져간다. 확보된 시간은 입력 검증·엣지케이스·교차검증 QA에 재배분한다.

- **제거**: 몬테카를로, 토네이도 차트, 확률 기반 리스크조정 랭킹
- **유지·강화**: NOI 빌드업, 현금흐름, Project/Equity IRR, NPV(Equity 기준), 핵심 비율, 1D/2D 민감도, 다중 딜 비교·스크리닝(결정론적)
- **v3 교정**: 할인율 일관성(Equity NPV), Equity Multiple 방어식, 포워드 NOI 명시계산, 입력 검증·엣지케이스 대폭 보강

## 1. 프로젝트 개요

여러 부동산 투자 후보(딜)를 한 번에 입력받아 수익성 지표를 산출하고, 투자 허들 통과 여부를 판정해 **Equity IRR 기준으로 랭킹**하는 의사결정 지원 웹 앱. 모든 계산은 결정론적이고 교차검증 가능하다.

- **핵심 채점 연결**: NOI 산출 → 현금흐름 → Project/Equity IRR → 민감도
- **데모 데이터**: 천안역세권 도시재생 리츠를 대표 예시 프리셋으로 탑재(원본 수치 강제 재현 금지, §11 참조)

## 2. 핵심 컨셉 & 차별점

| 구분 | 일반 가치평가 툴 | 딜 스크리너 |
|---|---|---|
| 분석 단위 | 단일 자산 | 다중 딜 동시 비교 |
| 의사결정 | "이 자산의 가치는?" | "어느 딜을 살까?" |
| 출력 | 지표 나열 | PASS/FAIL 판정 + IRR 랭킹 |
| 검증 | - | 엑셀 교차검증 + 엣지케이스 처리 |

차별점은 화려함이 아니라 **정합성**이다.

## 3. 사용자 흐름

1. 진입 → 천안역세권 프리셋이 기본 로드된 상태로 결과 표시
2. 딜 추가/편집(§6 필드) 및 §7 검증 통과
3. 투자 기준 설정: 목표 Equity IRR(허들) / 최소 DSCR / 최대 LTV
4. 자동 계산 → 비교 테이블에 핵심 지표 + PASS/FAIL + 순위
5. 딜 클릭 → 드릴다운: NOI 빌드업, 현금흐름 차트, 1D/2D 민감도

## 4. 기능 요구사항 (모듈별)

### M1. 딜 입력
- REQ-1.1 딜 추가/복제/삭제, 다중 딜 동시 보관(메모리 상태)
- REQ-1.2 입력 필드: §6 데이터 모델 전체
- REQ-1.3 OpEx 입력 방식: EGI 대비 %(`pct`) 또는 절대금액(`absolute`)
- REQ-1.4 대출 상환방식: 이자만(IO, 만기일시) / 원리금균등(연 단위). **데모 딜은 IO 사용**, 원리금균등은 구현하되 단위 테스트로 스폿 검증(잠재 결함 방지)
- REQ-1.5 모든 입력은 §7 검증을 통과해야 계산에 반영

### M2. 가치평가 엔진 (채점 핵심)
- REQ-2.1 연도별 NOI 빌드업(GPR→EGI→NOI), **n+1년차까지 명시 계산**(Exit 환원용)
- REQ-2.2 무차입/차입 현금흐름 프로젝션(보유기간 + Exit)
- REQ-2.3 Project IRR(무차입), Equity IRR(차입), **Equity NPV** 산출
- REQ-2.4 핵심 비율: Going-in Cap, 최소·1년차 DSCR, Cash-on-Cash, Equity Multiple
- REQ-2.5 IRR 솔버: Newton-Raphson + bisection 폴백. 수렴 실패·해 없음·**다중 부호변화 시 `irrValid=false`** 반환(임의값 금지)

### M3. 스크리닝 & 랭킹 (결정론적)
- REQ-3.1 PASS 조건: Equity IRR ≥ 허들 **AND** 최소 DSCR ≥ 기준 **AND** LTV ≤ 상한 **AND** `irrValid`
- REQ-3.2 랭킹: `irrValid` 플래그 우선(유효 먼저) → Equity IRR 내림차순. **매직넘버 미사용**
- REQ-3.3 PASS/FAIL 뱃지 및 미충족 사유 표시

### M4. 민감도 분석
- REQ-4.1 1D 테이블: 단일 변수(Exit Cap / 임대료 성장률 / 공실률 / 금리) → Equity IRR
- REQ-4.2 2D 히트맵: 두 변수 조합 → Equity IRR. 각 셀은 **전체 체인 재계산**, `irrValid=false` 셀은 회색+라벨

### M5. 시각화 & 프리셋
- REQ-5.1 다중 딜 비교 테이블(정렬·뱃지)
- REQ-5.2 드릴다운: NOI 빌드업 표 → 현금흐름 바차트 → 1D 테이블 → 2D 히트맵
- REQ-5.3 천안역세권 프리셋 + 빈 딜 템플릿

## 5. 계산 로직 명세 (정확성 = 채점 1순위)

표기: `t`=연차, `n`=보유기간, `g`=임대료 성장률, `v`=공실률, `i`=비용상승률
**모든 % 필드는 퍼센트포인트로 입력(5 = 5%), 내부에서 ÷100 후 사용.**

**NOI 빌드업 (t = 1 .. n+1 까지 계산)**
- `GPR_t = GPR_1 × (1+g)^(t-1)`
- `OtherIncome_t = OtherIncome_1 × (1+g)^(t-1)`  ← 기타수입도 g로 성장
- `EGI_t = GPR_t × (1 − v) + OtherIncome_t`
- OpEx: `absolute` 모드 → `OpEx_t = OpEx_1 × (1+i)^(t-1)` / `pct` 모드 → `OpEx_t = EGI_t × opex%` (이때 opexGrowth 무시)
- `NOI_t = EGI_t − OpEx_t`

**Exit (직접환원, 포워드 NOI — 명시 계산)**
- `TV = NOI_(n+1) / ExitCap`  (NOI_(n+1)은 위 빌드업으로 직접 산출, `×(1+g)` 근사 미사용)
- `NetSaleProceeds = TV × (1 − sellingCost%)`

**무차입 현금흐름 & Project IRR**
- `CF_0 = − Price × (1 + acqCost%)`
- `CF_t = NOI_t` (t = 1 .. n-1)
- `CF_n = NOI_n + NetSaleProceeds`
- `Project IRR = IRR(CF_0..n)`

**차입 (Levered)**
- `Loan = Price × LTV`, `Equity_0 = Price×(1+acqCost%) − Loan`
- IO: `DebtService_t = Loan × rate`, 만기 `LoanPayoff = Loan`
- 원리금균등(연 단위, 상환기간 N = amortTermYears):
  - `rate > 0`: `Payment = Loan × rate / (1 − (1+rate)^(−N))`
  - `rate = 0`: `Payment = Loan / N`
  - 잔액 `B_t` 표준식으로 산출. **N < n이면** 완납 후 DebtService=0, Exit LoanPayoff=0. **N ≥ n이면** LoanPayoff = `B_n`
- `LeveredCF_0 = − Equity_0`
- `LeveredCF_t = NOI_t − DebtService_t` (t = 1 .. n-1)
- `LeveredCF_n = NOI_n − DebtService_n + NetSaleProceeds − LoanPayoff`
- `Equity IRR = IRR(LeveredCF)`

**Equity NPV (할인율 = hurdleIRR)**
- `Equity NPV = Σ_{t=0..n} LeveredCF_t / (1 + hurdleIRR)^t`
- 무차입 흐름을 차입 허들로 할인하지 않는다(자본구조 일관성). Project NPV는 무차입 요구수익률 필드가 없어 범위 밖.

**핵심 비율**
- `Going-in Cap = NOI_1 / Price`
- `DSCR_t = NOI_t / DebtService_t` (보유기간 최소값 및 1년차 보고)
- `Cash-on-Cash = LeveredCF_1 / Equity_0`
- `Equity Multiple = Σ(LeveredCF 중 양수) / |Σ(LeveredCF 중 음수)|`  ← 캐피탈 콜 방어식

**IRR 솔버**
- Newton-Raphson(초기값 10%) → 미수렴 시 bisection(−99%~100%) 폴백
- 반복 상한 200, 허용오차 1e-7
- **사전 검사: LeveredCF 부호변화 횟수 > 1 이면 `irrValid=false`** (다중해 모호)
- 수렴 실패·해 없음 → `irrValid=false`, 숫자 반환 금지

## 7. 입력 검증 & 엣지케이스 (무결점 보증)

**검증 규칙**
- 모든 % 필드는 퍼센트포인트 입력 → 내부 ÷100
- 모든 수치 ≥ 0; `exitCap > 0`; `holdYears ≥ 1` 정수; `amortTermYears ≥ 1` 정수(amortizing 시)
- **LTV ∈ [0, 100%]**(대출은 자산가치 초과 불가) **AND `Equity_0 > 0` 직접 강제**. 후자는 "LTV 100% + 취득비 0%"의 단일 코너(자기자본=0)를 추가로 차단. 메시지: "LTV는 0~100%, 자기자본>0 필요"
- `hurdleIRR > −100%`
- 위반 시 계산 미실행 + 해당 필드 하이라이트

**엣지케이스**
- DebtService=0(LTV 0%) → DSCR=∞ 표기, FAIL 처리 안 함
- 분할상환 `rate=0` → `Payment=Loan/N`; `N<n` → 완납 후 DS=0·LoanPayoff=0
- IRR 미수렴/해 없음/다중 부호변화 → `irrValid=false`, "산출 불가" 표기, 랭킹 최하위, PASS 불가
- 2D 히트맵 계산 불가 셀 → 회색 + 라벨
- 0 나누기(Price=0, ExitCap=0 등) → 검증 단계 사전 차단
